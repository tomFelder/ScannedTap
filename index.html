SketchFab Viewer API Test

This test was designed to determine the flexibility of the SketchFab Viewer API for the purposes of our collaboration


This test is to be considered arguably successful. Whilst the API holds no ability to manipulate the navigation controls, a hack to persistently reinstate the pre-set camera via a short but smooth animation was implemented. One is able to rotate, zoom or pan away from the camera position but immediately the viewer will take back control and reinstate the original camera which is statically defined but unique cartesian coordinates.

Alternatively with HTML5 we can place an invisible iframe over the SketchFab viewer so the user has no ability to interaction with the viewer. In that circumstance I would argue; "Why bother?". We are using the viewer because it is 3D and without the ability to manipulate and interact with the 3D content the charm of 3D is lost.

In conclusion the former solution, implemented above, allows the user to marginally interact with 3D content. This interaction is heavily restricted to the dollhouse perspective we desire whilst also allowing the user to understand the content is being displayed to them live in 3D. An excellent result for such a limited API.
